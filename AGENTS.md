# AGENTS.md

## Project overview

Threads Country Badge is a Bun + TypeScript Chrome Manifest V3 extension. It annotates Threads authors with country flags or country names next to profile links.

## Required tooling

- Use Bun for package management and scripts.
- Use Oxlint only for linting. Do not add ESLint runtime/config unless explicitly requested.
- Use `@changepacks/cli` for release/version management.
- Use Husky pre-commit hooks to run `bun run lint` and `bun run test` before commits.

## Core commands

```bash
bun install
bun run typecheck
bun run lint
bun run test
bun run manifest:check
bun run build
bun run package
```

## Release flow

1. Create a changepack with `bun run changepacks`.
2. Commit the changepack log and push it to `main`.
3. Let the workflow run the quality gate, then `changepacks/action@v0.1.0` creates the `Update Versions` PR.
4. Merge the version PR after review; the action creates the GitHub Release, and the Chrome Web Store job publishes when `package.json` appears in the changepacks output.
5. Use `bun run changepacks:update` locally only when previewing or manually applying a release version.

## Chrome Web Store deployment

- Deployment workflow: `.github/workflows/publish-chrome-web-store.yml` (`Changepacks & Chrome Web Store`).
- Required GitHub Secrets:
  - `CHROME_EXTENSION_ID`
  - `GOOGLE_CLIENT_ID`
  - `GOOGLE_CLIENT_SECRET`
  - `GOOGLE_REFRESH_TOKEN`
- Store listing, privacy declarations, and reviewer-facing administrative fields are documented in `docs/deployment.md`.

## Testing and coverage

- `bunfig.toml` enables coverage and requires 100% coverage for files included in Bun's coverage set.
- Keep tests inside layer-local `__tests__/` directories: use `src/__tests__/` for `src/*.ts` modules and `src/shared/__tests__/` for `src/shared/*.ts` modules.
- Keep pure logic in small modules where possible so it remains easy to cover without browser automation.

## Extension constraints

- `src/manifest.json` is the source manifest; `dist/` is generated by `bun run build` and must not be committed.
- The MAIN-world `src/injected.ts` path is required for Threads request/template capture and should not be moved to a service worker without re-validating browser behavior.
- The extension depends on the user's existing Threads browser session; it must not handle login credentials.
- Chrome Web Store rejects duplicate manifest versions, so every release must bump the version.
- Chrome Web Store uploads should use the zip generated by `bun run package`; do not generate or commit `.crx`/`.pem` files for store publishing.

## Verification expectations

- For code changes, run `bun run typecheck`, `bun run lint`, `bun run test`, and `bun run build`.
- For release changes, additionally run `bun run manifest:check`.
- For live behavior changes, use the CDP/Chromium scripts after rebuilding and relaunching the extension.
